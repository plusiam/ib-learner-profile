## 1. 프로젝트 개요

### 1.1 기본 정보

| 항목 | 내용 |
|---|---|
| **프로젝트명** | IB 학습자상 탐색 웹앱 |
| **앱 이름(안)** | "나는 어떤 학습자일까?" |
| **저장소명(안)** | `ib-learner-profile` |
| **GitHub 계정** | plusiam |
| **배포** | GitHub Pages + Vercel 병행 |
| **기술 스택** | HTML + CSS + Vanilla JS (단일 index.html) |
| **대상** | 초등학교 5학년 (10~11세) |
| **소요 시간** | 12~15분 |

### 1.2 교육적 맥락

| 항목 | 내용 |
|---|---|
| **수업 연계** | 5학년 도덕 2단원 "반성과 성찰로 성장하는 나" |
| **성취기준** | [6도01-02] 자신의 생활을 반성하고 성찰하는 생활의 중요성 탐구 |
| **상위 프로젝트** | 그림책 작가 프로젝트 (IB 학습자상 기반 캐릭터 → 그림책 창작) |
| **앱의 역할** | 프로젝트 1단계 — 자기 탐색을 통한 학습자상 발견 |

### 1.3 학습 흐름에서의 위치

```
[1단계] 이 앱 — IB 학습자상 TOP 3 탐색
    ↓
[2단계] 학습자상 기반 캐릭터 설정 (이름, 성격, 고민, 미덕)
    ↓
[3단계] 그림책 스토리 구성 (story-worksheet 활용)
    ↓
[4단계] 그림책 완성 + 나눔 (발표/전시)
```

### 1.4 기존 시리즈 앱과의 관계

```
dream-ingredients  →  dream-type(홀랜드)  →  ib-learner-profile(이 앱)  →  story-worksheet
 (꿈 재료 카드)       (흥미 유형 검사)        (학습자상 탐색)                (그림책 스토리보드)
```

## 2. 앱 성격 정의

### 2.1 이 앱은 "검사 도구"가 아니라 "탐색 활동"이다

| 구분 | 표준화 검사 | **이 앱 (탐색형 활동)** |
|---|---|---|
| 목적 | 정확한 진단·분류 | **자기 이해의 출발점, 대화의 씨앗** |
| 신뢰도 요구 | 높음 (α ≥ .70) | 적정 수준 (방향성 파악) |
| 결과 활용 | 배치·판정 | **그림책 캐릭터 설정 연결** |
| 결과 메시지 톤 | 확정적 | **탐색적 ("이런 면이 있구나!")** |

> **핵심 원칙**: 결과 화면에 반드시 "정답은 없어! 모든 학습자상이 소중해. 이건 참고용이야, 네 마음이 제일 중요해!" 메시지를 포함한다.

## 3. 화면 흐름 (User Flow)

```
[인트로 화면]
    ↓ 이름 입력 + "시작하기" 버튼
[설문 화면] — 30문항 순차 진행
    ↓ 프로그레스 바 표시 (1/30 ~ 30/30)
    ↓ 문항 유형별 다른 UI 렌더링
    ↓ 이전/다음 버튼 네비게이션
[임시 결과 화면] — "너의 TOP 3 학습자상이야!"
    ↓
[확인 질문 화면] — "이 중에서 가장 되고 싶은 학습자상은?"
    ↓ TOP 3 중 선택 (또는 "다른 학습자상 고르기" 옵션)
[최종 결과 카드 화면]
    ↓ 학습자상 설명 + 미덕 연결 + 그림책 캐릭터 힌트
    ↓ 이미지 저장 / 다시 하기
```

## 4. 문항 설계

### 4.1 문항 구조 요약

| 항목 | 내용 |
|---|---|
| **총 문항 수** | 30문항 + 확인 질문 1개 |
| **학습자상당 문항** | 3개 |
| **점수 체계** | 각 문항 1~4점 → 학습자상별 최대 12점 |
| **문항 순서** | 랜덤 셔플 (같은 학습자상 연속 방지) |

### 4.2 문항 유형 (4가지 혼합)

#### 유형 ① 리커트 (8문항)

- 형태: "나는 ~하는 편이다" + 4점 척도
- 선택지: 전혀 아니다(1) / 별로 아니다(2) / 조금 그렇다(3) / 매우 그렇다(4)
- UI: 이모지 슬라이더 또는 4개 버튼

```
예시)
"새로운 것을 알게 되면 더 깊이 파고들고 싶다."
😑 전혀 아니다 | 🤔 별로 아니다 | 😊 조금 그렇다 | 🤩 매우 그렇다
→ 탐구하는 사람에 1~4점 부여
```

#### 유형 ② 카드 선택 (7문항)

- 형태: 이모지+짧은 문장이 담긴 카드 4장 중 가장 끌리는 1장 선택
- 점수: 선택한 카드에 해당하는 학습자상에 4점, 나머지는 0점
- UI: 가로 또는 2×2 그리드 카드 레이아웃

```
예시)
"자유시간이 생기면 나는?"
🔍 모르는 것 찾아보기    🎨 그림/만들기
🤝 친구와 이야기하기     📖 조용히 생각 정리하기
→ 각각 탐구형/도전형/소통형/성찰형에 4점
```

#### 유형 ③ A vs B (7문항)

- 형태: 두 가지 성향 중 더 가까운 쪽 선택 + 강도 표시
- 점수: 선택한 쪽에 3~4점 (강도에 따라), 반대쪽에 1~2점
- UI: 양쪽 카드 + 중간 슬라이더 또는 "조금 A / 많이 A / 조금 B / 많이 B" 4단계

```
예시)
A: "규칙을 잘 지키는 게 중요해" ⚖️
B: "새로운 방법을 시도하는 게 중요해" 🚀
→ A 선택 시 원칙형에 가점, B 선택 시 도전형에 가점
```

#### 유형 ④ 상황 제시 (8문항)

- 형태: "이럴 때 나는?" 짧은 시나리오 + 4개 행동 선택지
- 점수: 각 선택지가 서로 다른 학습자상에 대응, 선택한 것에 4점
- UI: 상황 박스 + 4개 선택지 카드

```
예시)
"모둠 활동에서 의견이 다른 친구가 있어. 나는..."
A: "왜 그렇게 생각하는지 이유를 물어본다" → 탐구하는
B: "그 친구의 말을 끝까지 들어본다" → 열린 마음
C: "내 생각을 차근차근 설명해본다" → 소통하는
D: "서로 양보할 점을 찾아본다" → 배려하는
→ 선택한 항목의 학습자상에 4점
```

### 4.3 학습자상별 문항 유형 배정

| 학습자상 | 문항1 유형 | 문항2 유형 | 문항3 유형 |
|---|---|---|---|
| 탐구하는 (Inquirers) | 리커트 | 상황제시 | 카드선택 |
| 지식이 풍부한 (Knowledgeable) | 카드선택 | A vs B | 리커트 |
| 사고하는 (Thinkers) | 상황제시 | 리커트 | A vs B |
| 소통하는 (Communicators) | A vs B | 카드선택 | 상황제시 |
| 원칙을 지키는 (Principled) | 리커트 | 상황제시 | A vs B |
| 열린 마음 (Open-minded) | 카드선택 | A vs B | 상황제시 |
| 배려하는 (Caring) | 상황제시 | 리커트 | 카드선택 |
| 도전하는 (Risk-takers) | A vs B | 카드선택 | 리커트 |
| 균형 잡힌 (Balanced) | 리커트 | 상황제시 | A vs B |
| 성찰하는 (Reflective) | 상황제시 | A vs B | 카드선택 |

### 4.4 문항 작성 원칙

- 5학년 눈높이의 **쉬운 어휘** 사용 (초등 국어 교과서 수준)
- 학교생활, 친구관계, 가정생활 등 **학생 일상 맥락**의 시나리오
- 이모지 적극 활용으로 **시각적 친근감** 확보
- 특정 학습자상이 "정답"처럼 보이지 않도록 **선택지 균형** 유지
- 성별·문화적 편향이 없는 **중립적 표현** 사용

## 5. 점수 산출 및 결과 로직

### 5.1 점수 체계

```javascript
// 학습자상별 점수 객체
scores = {
  INQ: 0,  // 탐구하는 (Inquirers) — 최대 12점
  KNO: 0,  // 지식이 풍부한 (Knowledgeable)
  THI: 0,  // 사고하는 (Thinkers)
  COM: 0,  // 소통하는 (Communicators)
  PRI: 0,  // 원칙을 지키는 (Principled)
  OPE: 0,  // 열린 마음 (Open-minded)
  CAR: 0,  // 배려하는 (Caring)
  RIS: 0,  // 도전하는 (Risk-takers)
  BAL: 0,  // 균형 잡힌 (Balanced)
  REF: 0   // 성찰하는 (Reflective)
}
```

### 5.2 유형별 점수 부여 방식

| 유형 | 점수 부여 |
|---|---|
| 리커트 | 1~4점 직접 부여 |
| 카드 선택 | 선택한 카드의 학습자상에 4점 |
| A vs B | "많이 A"=4, "조금 A"=3, "조금 B"=3, "많이 B"=4 (각각 해당 학습자상에) |
| 상황 제시 | 선택한 행동의 학습자상에 4점 |

### 5.3 TOP 3 추출

```
1. 학습자상별 합산 점수 계산
2. 점수 내림차순 정렬
3. 상위 3개 추출
4. 동점 시 처리: 상황제시형 문항 점수 우선 반영
```

### 5.4 확인 질문 반영

```
1. 임시 TOP 3 제시
2. 학생이 "가장 되고 싶은 학습자상" 1개 선택
   - TOP 3 중 선택 → 해당 학습자상이 최종 TOP 1
   - "다른 학습자상 고르기" 선택 → 10개 중 선택 가능 → 해당이 TOP 1, 나머지 TOP 2·3 유지
3. 최종 결과: 선택한 TOP 1 + 기존 TOP에서 재배치한 TOP 2·3
```

### 8.1 디자인 톤

| 항목 | 방향 |

| **컬러** | 각 학습자상별 지정 색상 활용 (6.1 데이터 참조) |
| **배경** | 따뜻한 크림색 (#FFF9F0) 계열 |
| **카드** | 둥근 모서리 (border-radius: 16~20px), 부드러운 그림자 |
| **폰트** | Nanum Round Square (본문) + Gaegu (포인트/손글씨 느낌) |
| **이모지** | 적극 활용 — 각 학습자상, 문항 유형, 진행 상태 표시 |

### 8.2 반응형 설계

| 디바이스 | 기준 너비 | 비고 |
|---|---|---|
| 태블릿 (주 사용 환경) | 600~1024px | 최적화 대상 |
| 모바일 | ~599px | 세로 스크롤 대응 |
| 데스크톱 | 1025px~ | max-width: 600px 중앙 정렬 |

### 8.3 접근성

- 충분한 터치 영역 (최소 44px × 44px)
- 색상 대비 4.5:1 이상
- 문항 텍스트 크기 최소 16px
- 진행 상태 명확히 표시 (프로그레스 바 + 숫자)

## 9. 기술 사양

### 9.1 파일 구조

```
ib-learner-profile/
├── index.html          # 단일 파일 (HTML + CSS + JS 통합)
└── README.md           # 프로젝트 설명
```

### 9.2 외부 의존성

| 라이브러리 | 용도 | CDN |
|---|---|---|
| Google Fonts | Nanum Round Square, Gaegu | fonts.googleapis.com |
| html2canvas | 결과 카드 이미지 저장 | cdnjs.cloudflare.com |

### 9.3 코드 규칙

- **모든 주석은 한글로** 작성
- 변수·함수명은 camelCase 영문, 주석으로 한글 설명
- CSS 변수(`:root`)로 색상·크기 관리
- 문항 데이터는 JS 객체 배열로 코드 상단에 분리 정의
- 학습자상 데이터(PROFILES)도 JS 객체로 상단 분리 정의

## 10. 개발 순서

### Phase 1: 문항 콘텐츠 작성
1. 학습자상별 3문항 × 10개 = 30문항 텍스트 작성
2. 카드선택형 선택지 4개씩 작성
3. 상황제시형 시나리오 + 선택지 작성
4. A vs B 쌍 구성
5. 문항별 점수 매핑 검증

### Phase 2: 기본 구조 구현
1. HTML 스크린 구조 (인트로/설문/확인/결과)
2. CSS 기본 레이아웃 + 색상 변수
3. 문항 데이터 JS 객체 정의
4. 학습자상 데이터(PROFILES) 정의

### Phase 3: 설문 기능 구현
1. 문항 셔플 로직
2. 4가지 문항 유형별 렌더러
3. 응답 저장 + 이전/다음 네비게이션
4. 프로그레스 바

### Phase 4: 결과 화면 구현
1. 점수 계산 + TOP 3 추출
2. 확인 질문 UI + 로직
3. 결과 카드 레이아웃
4. 레이더 차트 (Canvas)
5. 학습자상 설명 + 미덕 섹션
6. 그림책 캐릭터 힌트

### Phase 5: 마무리
1. html2canvas 이미지 저장
2. 반응형 테스트 (태블릿/모바일)
3. 접근성 점검
4. 문항 밸런스 테스트 (편향 체크)
5. GitHub Pages + Vercel 배포
